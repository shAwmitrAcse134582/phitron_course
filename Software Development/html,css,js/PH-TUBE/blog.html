Ques_1: Discuss the scope of var, let and const.

Answer:

Var: The region in which these variables can be utilized is referred to as the scope. Function/locally, globally, or both can be the scope of a var declaration.The scope is global when a var variable is declared outside of a function. This implies that any variable declared using the var keyword outside of a function block is available across the whole window.A variable has function scope when it is declared inside of a function. This suggests that it is open to use and restricted to that purpose.


Let: Let isÂ a block scope. A block is a section of code that is bordered by. Furthermore, a variable declared in a block using the "let" keyword is only used inside that block.


Const: The scope of a variable declared with the const keyword is constrained to the block (a function or a block), just as the scope of let declarations. The primary distinction is that they are immutable, i.e., their value doesn't change inside the scope despite updates or re-declarations.

Ques_2: Tell us the use cases of null and undefined.

Answer:

Undefined: A variable's value will be undefined if it is declared but has not yet been given a value.

Use cases of undefined:

  • When defining a funtion with parametres but not passing value for a parametre,it's automatically undefined
  • If any variable is decalred without an initial value will beundefined
  • Accesing object property that doesn't exist

Null: Null is a special value that stands in for an empty or unknown value in JavaScript.

Use cases of null:

  • A variable that has no value can be identified as null.
  • Can be reassigned with any value that was at first initialized with null.
  • Sometimes function returns null to indicate that they don't succeed in their operation.
Ques_3: What do you mean by REST API?

Answer:

Since application programming interfaces (APIs) exist in a variety of forms and sizes, it can be challenging for beginners to comprehend what they are and how to utilize them.There are many types of API in which REST API is one of the most used.

Web APIs are developed using the Representational State Transfer (REST) software architecture style. Because they may be used to make data, information, algorithms, media, and other digital resources accessible through web URLs, REST APIs offer straightforward, consistent interfaces. REST APIs are essentially the ones that are utilized the most on the web nowadays.

There are six guiding restrictions that must be met for the API service to be RESTful:

  • Use of a uniform interface
  • Stateless operations
  • Client-server based
  • RESTful resource caching
  • Code on demand
  • Layerd system
